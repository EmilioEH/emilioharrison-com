# Core Library Conventions

## Nanostores

- All shared state lives in `src/lib/` as `*Store.ts` files
- Use `atom()` for simple values, `map()` for objects
- Use `@nanostores/persistent` for data that survives refresh (user preferences, UI state)
- Export typed store and convenience actions — don't expose raw `.set()` calls

## Types

- All shared types live in `src/lib/types.ts`
- `Recipe`, `Ingredient`, `StructuredIngredient`, `StructuredStep` are the core domain types
- **Firestore constraint**: No nested arrays (`number[][]`). Use `Array<{ indices: number[] }>` instead.

## Firebase

- **Client** (`firebase-client.ts`): Browser-only Firestore/Auth operations
- **Server** (`firebase-server.ts`): Server-side admin SDK, accessed via `getRequestContext()`
- **REST** (`firebase-rest.ts`): Service Account REST calls for storage uploads
- **Never import firebase-admin in browser code**

## Utilities

- Date helpers in `date-helpers.ts` — use these instead of raw `Date` manipulation
- Type guards in `type-guards.ts` — use for runtime validation
- API utilities in `api-utils.ts` — standard fetch wrappers with error handling
- Grocery logic in `grocery-logic.ts` and `grocery-utils.ts`

## Grocery Categories (fixed order)

Produce → Meat → Dairy → Bakery → Frozen → Pantry → Spices → Other
