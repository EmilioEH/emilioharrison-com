# Service Layer Conventions

## Gemini AI Integration

- **SDK**: `@google/genai` — initialize with `initGeminiClient()` from `api-helpers.ts`
- **Model**: `gemini-2.0-flash` for fast parsing, `gemini-2.0-pro-exp` for complex reasoning
- **Always use structured output**:
  ```ts
  response_mime_type: 'application/json'
  response_schema: {
    /* your schema */
  }
  ```

## AI Parser (`ai-parser.ts`)

Two parsing modes — never mix them:

- `style='strict'`: Initial import — transcribe exactly what the user provided, no embellishment
- `style='enhanced'`: Background enhancement — full Kenji-style restructuring with macro-steps, sensory endpoints, dual measurements

**Never force enhancement on initial import.** Users expect their recipe to look like what they pasted.

## Error Handling

- Wrap all external API calls in try/catch
- Return structured error objects, not thrown exceptions
- Log errors server-side with context (what was attempted, what failed)
- Return user-friendly fallback messages

## Adding New Services

1. Create in `src/lib/services/`
2. Export a clear public API — hide implementation details
3. Services should be stateless — use stores for state
4. Services are server-side only unless explicitly needed client-side
