# API Route Conventions

## Request Context

Every API route runs on Cloudflare Workers. Middleware calls `setRequestContext(context)` before your handler runs. Access it via:

```ts
import { getRequestContext } from '../../lib/request-context'
```

## Rules

- **Never use Firebase Client SDK** (`firebase/storage`) server-side — use `FirebaseRestService` with Service Account
- **Always validate input** before processing
- **Return proper HTTP status codes** (200, 400, 401, 404, 500)
- **Never expose internal error messages** to the client — log them server-side, return user-friendly messages
- **Use `initGeminiClient()`** from `api-helpers.ts` for any Gemini AI calls
- **File uploads** go through `POST /api/uploads` only — never accept file data in other endpoints
