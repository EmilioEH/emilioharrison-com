# Component Conventions

## UI Library

Use **shadcn/ui** components (Radix + Tailwind). Available primitives in `src/components/ui/`:

- `button.tsx`, `card.tsx`, `dialog.tsx`, `sheet.tsx`, `tabs.tsx`
- `input.tsx`, `textarea.tsx`, `label.tsx`, `checkbox.tsx`
- `dropdown-menu.tsx`, `badge.tsx`, `separator.tsx`
- `ResponsiveModal.tsx`, `Fab.tsx`, `StarRating.tsx`, `ImageViewer.tsx`

## Layout Primitives (ALWAYS use these)

Import from `src/components/ui/layout.tsx`:

```tsx
<Stack spacing="lg">...</Stack>     // Vertical stack
<Inline spacing="sm">...</Inline>   // Horizontal row
<Cluster spacing="xs">...</Cluster> // Wrapping flex items
```

Spacing: `xs`(2px) | `sm`(8px) | `md`(16px) | `lg`(24px) | `xl`(32px) | `2xl`(48px)

**Never use** raw `space-y-*` or `gap-*` for layout — use the primitives above.

## Positioning

- Use CSS variables: `top-header`, `top-content-top`, `pt-content-top`
- **Never hardcode pixel values** for `top`, `padding-top` on sticky elements
- Use `data-*` attributes to override CSS variables for state-based layout

## Mobile-First Modals

```tsx
className = 'fixed left-4 right-4 mx-auto ... sm:left-1/2 sm:-translate-x-1/2'
```

## Animations

Use **Framer Motion** for transitions and animations. Keep them subtle and purposeful.

## State Management

- Use **nanostores** from `src/lib/` — never create local React context for shared state
- Use `@nanostores/persistent` for anything that should survive page refresh
- Use `useStore()` from `@nanostores/react` to subscribe in components

## SPA Routing

- **Never create new Astro pages** for app features
- Add new views as `ViewMode` in `RecipeManager.tsx`
- Use `useRouter` hook for navigation, not `<a>` tags or Astro routing
