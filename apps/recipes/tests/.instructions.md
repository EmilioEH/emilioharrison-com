# Testing Conventions

## E2E Tests (Playwright)

- Base URL: `http://127.0.0.1:8788` — always use `127.0.0.1`, not `localhost` (auth cookies require it)
- Tests run against `preview:wrangler` — build with `npm run build:test` first (`PUBLIC_TEST_MODE=true`)
- All mock data and fixtures come from `tests/msw-setup.ts`

## MSW Setup (`msw-setup.ts`)

- Provides `test` fixture with `mockPage` that auto-intercepts API routes
- Includes test recipe archetypes: simple (2-3 steps), medium (5-7 steps), complex (10+ steps with timers)
- Auth cookies are auto-set by the fixture
- Use `mockPage.goto('/')` to start — routes are already mocked

## Writing a New E2E Test

```ts
import { test, expect } from './msw-setup'

test.describe('Feature Name', () => {
  test('should [user action] and [expected result]', async ({ mockPage }) => {
    await mockPage.goto('/')
    // Interact with the UI as a user would
    await mockPage.getByRole('button', { name: 'Something' }).click()
    // Assert what the user sees
    await expect(mockPage.getByText('Expected result')).toBeVisible()
  })
})
```

## Unit Tests (Vitest)

- Co-located with source: `src/lib/something.test.ts`
- Use `@testing-library/react` for component tests
- Run with `npm run test:unit`

## Rules

- Every new UI feature MUST have an E2E test
- Test user behavior, not implementation details
- Use descriptive test names: "should [action] when [condition]"
- Use `getByRole`, `getByText`, `getByLabel` — avoid `getByTestId` unless necessary
